## 1. 변동성 돌파 (Volatility Breakout)

> 파일: `volatility_breakout.py` | 시장: **상승장** | 난이도: 쉬움

### 원리

래리 윌리엄스(Larry Williams)가 개발한 단기 트레이딩 전략. 전일 가격 변동폭의 일정 비율(K)만큼 당일 시가에서 상승하면, 강한 상승 모멘텀으로 판단하여 매수한다.

**한국 코인 트레이딩에서 가장 인기 있는 전략**으로, 비트코인에서 역사적으로 좋은 성과를 보였다.

### 매수/매도 조건

```
매수: 현재가 > 당일 시가 + (전일 고가 - 전일 저가) × K

매도:
  - 손절: 매수가 대비 -5% 이하
  - 트레일링 스탑: 매수 이후 최고가 대비 -3% 이하
```

### 시각화

```
가격
 │
 │                          ╱ ← 현재가 (돌파!)
 │                        ╱
 │   ─ ─ ─ ─ ─ ─ ─ ─ ─ ─    ← 돌파 기준선 = 시가 + 레인지 × K
 │                 ╱
 │ ──────────────╱            ← 당일 시가
 │
 │  │← 전일 레인지 →│
 │  ┌──────────────┐
 │  │   전일 저가~고가   │
 │  └──────────────┘
 └──────────────────────── 시간
```

### 파라미터

| 파라미터 | 기본값 | 범위 | 설명 |
|---------|-------|------|------|
| `k_value` | 0.5 | 0.3~0.7 | K값이 낮을수록 공격적 (진입 빈번), 높을수록 보수적 |
| `stop_loss_pct` | -5.0% | -3~-7% | 손절 기준 |
| `trailing_stop_pct` | -3.0% | -2~-5% | 트레일링 스탑 기준 |

### 시장 평가

- **강점**: 구현 간단, 코인 시장의 높은 일중 변동성을 활용, 백테스트 성과 양호
- **약점**: K값에 따라 성과 차이가 큼, 하락장에서 연속 손실 가능, 수수료 누적 주의
- **최적 시장**: 상승 추세가 명확한 시장, 변동성이 높은 코인 (BTC, ETH)
- **부적합 시장**: 횡보장 (허위 돌파 빈번), 하락장

## 2. 이동평균 교차 (MA Crossover)

> 파일: `ma_crossover.py` | 시장: **추세장** | 난이도: 쉬움

### 원리

가장 고전적인 추세 추종 전략. 단기 이동평균선이 장기 이동평균선을 상향 돌파하면 상승 추세 시작(골든크로스), 하향 돌파하면 하락 추세 시작(데드크로스)으로 판단한다.

### 매수/매도 조건

```
매수: MA(5)가 MA(20)를 위로 돌파 (골든크로스)
매도: MA(5)가 MA(20)를 아래로 돌파 (데드크로스)
```

### 시각화

```
가격
 │
 │        MA(5) ↗          ← 단기 이동평균 (빠른 반응)
 │       ╱    ╲
 │   ──╳──────╳──          ← MA(20) 장기 이동평균
 │    ╱↑         ↑╲
 │   ╱ 골든크로스  데드크로스
 │  ╱  (매수!)    (매도!)
 └──────────────────────── 시간
```

### 파라미터

| 파라미터 | 기본값 | 범위 | 설명 |
|---------|-------|------|------|
| `short_period` | 5 | 5~20 | 단기 이동평균 기간 |
| `long_period` | 20 | 20~60 | 장기 이동평균 기간 |

### 시장 평가

- **강점**: 구현 매우 간단, 큰 추세를 확실히 잡을 수 있음, 이해하기 쉬움
- **약점**: 신호 지연(lag)이 큼, 횡보장에서 잦은 허위 신호(whipsaw) 발생
- **최적 시장**: 명확한 상승/하락 추세가 있는 시장
- **부적합 시장**: 횡보장 (골든/데드크로스가 반복되면서 손실 누적)

## 3. MACD

> 파일: `macd_strategy.py` | 시장: **추세장** | 난이도: 쉬움

### 원리

이동평균 교차의 발전형. 단순 MA 대신 EMA(지수이동평균)를 사용하고, MACD 라인과 시그널 라인의 교차로 매매 시점을 판단한다. 히스토그램으로 추세 강도까지 파악 가능.

```
MACD 라인    = EMA(12) - EMA(26)
시그널 라인   = MACD의 EMA(9)
히스토그램   = MACD - 시그널
```

### 매수/매도 조건

```
매수: MACD 라인이 시그널 라인을 상향 돌파 (0선 위면 더 강한 신호)
매도: MACD 라인이 시그널 라인을 하향 돌파
```

### 시각화

```
MACD
 │
 │    MACD 라인 ↗
 │      ╱
 ─ ── ╳ ─ ─ ─ ─ ─ ─ ─ ─ ─  ← 0선
 │  ╱ ↑
 │ ╱  시그널 돌파 (매수!)
 │╱
 │      시그널 라인 ──
 │
 │  ▓▓▓                      ← 히스토그램 (양수 = 상승 모멘텀)
 └──────────────────────── 시간
```

### 파라미터

| 파라미터 | 기본값 | 설명 |
|---------|-------|------|
| `fast` | 12 | 빠른 EMA 기간 |
| `slow` | 26 | 느린 EMA 기간 |
| `signal_period` | 9 | 시그널 EMA 기간 |

### 시장 평가

- **강점**: 단순 MA 교차보다 신뢰도 높음, 추세 강도까지 판단 가능, 다이버전스 활용 가능
- **약점**: 횡보장에서 여전히 허위 신호, 파라미터 최적화 필요
- **최적 시장**: 추세가 뚜렷한 시장 (상승장/하락장)
- **부적합 시장**: 좁은 범위의 횡보장

## 4. 슈퍼트렌드 (Supertrend)

> 파일: `supertrend.py` | 시장: **추세장** | 난이도: 중간

### 원리

ATR(평균진폭)을 기반으로 동적 지지/저항선을 생성. 가격이 이 라인 위에 있으면 상승 추세, 아래면 하락 추세로 판단한다. **변동성에 자동으로 적응**하기 때문에 변동성이 높은 코인 시장에 적합하다.

### 매수/매도 조건

```
매수: 슈퍼트렌드 방향이 하락→상승으로 전환
매도: 슈퍼트렌드 방향이 상승→하락으로 전환
```

### 시각화

```
가격
 │        ╱╲
 │       ╱  ╲      ╱╲
 │      ╱    ╲    ╱  ╲ ╱
 │  ═══╱══════╲══╱═══        ← 슈퍼트렌드 라인 (상승=지지선)
 │  ↑           ╲╱
 │  상승 전환     ↑
 │  (매수!)      하락 전환
 │               (매도!)
 └──────────────────────── 시간
```

### 파라미터

| 파라미터 | 기본값 | 범위 | 설명 |
|---------|-------|------|------|
| `st_period` | 10 | 7~14 | ATR 계산 기간 |
| `st_multiplier` | 3.0 | 2.0~4.0 | ATR 배수 (클수록 보수적) |

### 시장 평가

- **강점**: 변동성 적응형이라 코인에 적합, MA 교차보다 허위 신호 적음, 트레일링 스탑 역할도 겸함
- **약점**: 횡보장에서 수익성 낮음, 급격한 추세 전환 시 큰 손실
- **최적 시장**: 강한 추세가 형성되는 구간
- **부적합 시장**: 장기 횡보장

## 5. RSI 평균 회귀 (RSI Mean Reversion)

> 파일: `rsi_mean_reversion.py` | 시장: **횡보장** | 난이도: 쉬움

### 원리

RSI(상대강도지수)가 극단값에 도달하면 가격이 평균으로 되돌아올 가능성이 높다는 가정. 과매도 구간에서 반등하면 매수, 과매수 구간에서 하락하면 매도한다.

**주의**: 강한 추세에서는 RSI가 극단값에서 더 진행할 수 있어 위험하다 (떨어지는 칼날 잡기). 반드시 횡보장에서만 사용해야 한다.

### 매수/매도 조건

```
매수: RSI가 30 이하로 하락 후 30 위로 복귀 (과매도 반등)
매도: RSI가 70 이상으로 상승 후 70 아래로 복귀 (과매수 하락)
```

### 시각화

```
RSI
 │
70├─ ─ ─ ─ ─ ╲─ ─ ─ ─ ─ ─  ← 과매수 영역
 │            ╲↓ 매도!
50├────────────────────────
 │      ↑ 매수!
30├─ ─ ╱─ ─ ─ ─ ─ ─ ─ ─ ─  ← 과매도 영역
 │  ──╱
 └──────────────────────── 시간
```

### 파라미터

| 파라미터 | 기본값 | 범위 | 설명 |
|---------|-------|------|------|
| `rsi_period` | 14 | 7~21 | RSI 계산 기간 |
| `oversold` | 30 | 20~35 | 과매도 기준 |
| `overbought` | 70 | 65~80 | 과매수 기준 |

### 시장 평가

- **강점**: 구현 매우 간단, 횡보장에서 높은 승률, 과매수/과매도 구간이 명확
- **약점**: 강한 추세에서 치명적 손실 가능, 코인은 RSI 30 이하에서도 계속 하락하는 경우 빈번
- **최적 시장**: 명확한 추세 없이 등락을 반복하는 횡보장
- **부적합 시장**: 상승장/하락장 (추세에 역행하는 매매 위험)

## 6. 볼린저 밴드 (Bollinger Bands)

> 파일: `bollinger_bands.py` | 시장: **횡보장** | 난이도: 쉬움

### 원리

가격이 통계적으로 평균(중심선)으로 회귀하는 성질을 이용. 상단/하단 밴드(평균 ± 2σ)를 이탈한 후 다시 밴드 안으로 복귀하면 반전 진입한다.

### 매수/매도 조건

```
매수: 가격이 하단 밴드 아래로 이탈 후 밴드 안으로 복귀
매도: 가격이 상단 밴드 위로 이탈 후 밴드 안으로 복귀
      또는 중심선(SMA 20) 도달 시 익절
```

### 시각화

```
가격
 │
 │ ╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌   ← 상단 밴드 (SMA + 2σ)
 │                   ╱↓ 매도!
 │  ────────────────╱────   ← 중심선 (SMA 20) → 익절 목표
 │        ╱╲       ╱
 │       ╱  ╲     ╱
 │ ╌╌╌╌╌╌╌╌╌╲╌╌╌╱╌╌╌╌╌╌╌   ← 하단 밴드 (SMA - 2σ)
 │            ╲╱
 │            ↑ 매수!
 └──────────────────────── 시간
```

### 파라미터

| 파라미터 | 기본값 | 범위 | 설명 |
|---------|-------|------|------|
| `bb_period` | 20 | 15~25 | 이동평균 기간 |
| `bb_std` | 2.0 | 1.5~2.5 | 표준편차 배수 |

### 시장 평가

- **강점**: 직관적, 변동성을 자동 반영, 횡보장에서 높은 승률
- **약점**: 강한 추세에서 역추세 진입하면 큰 손실, 밴드 폭 급변 시 불안정
- **최적 시장**: 레인지 바운드 (지지/저항이 명확한 횡보장)
- **부적합 시장**: 강한 일방향 추세 (밴드 이탈이 실제 추세일 수 있음)

## 7. 그리드 트레이딩 (Grid Trading)

> 파일: `grid_trading.py` | 시장: **횡보장** | 난이도: 중간

### 원리

가격 범위를 일정 간격의 격자(그리드)로 나누고, 가격이 하락하면 매수, 상승하면 매도를 반복. **추세를 예측하지 않는 전략**으로, 24시간 운영되는 코인 시장에 특히 적합하다.

### 매수/매도 조건

```
매수: 가격이 하락하여 더 낮은 격자에 진입할 때
매도: 매수 격자보다 한 칸 이상 위 격자에 도달할 때
```

### 시각화

```
가격
 │
 │ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  격자 #10 (상한)
 │            ╱↓ 매도!
 │ ─ ─ ─ ─ ╱─ ─ ─ ─ ─ ─ ─  격자 #8
 │         ╱
 │ ─ ─ ─ ╱─ ─ ─ ─ ─ ─ ─ ─  격자 #6
 │       ╱  ╲
 │ ─ ─ ╱─ ─ ╲─ ─ ─ ─ ─ ─   격자 #4
 │    ╱       ╲
 │ ─╱─ ─ ─ ─ ─╲─ ─ ─ ─ ─   격자 #2 ← 매수!
 │               ╲╱
 │ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  격자 #0 (하한)
 └──────────────────────── 시간
```

### 파라미터

| 파라미터 | 기본값 | 범위 | 설명 |
|---------|-------|------|------|
| `grid_count` | 10 | 5~20 | 격자 수 (많을수록 잦은 매매) |
| `range_pct` | 10.0% | 5~20% | 가격 범위 (중심가 대비 ±%) |

### 시장 평가

- **강점**: 추세 예측 불필요, 자동화 용이, 횡보장에서 꾸준한 수익, 감정 개입 없음
- **약점**: 강한 일방향 추세 시 큰 손실 (하락장에서 물림), 자금 효율성 낮음
- **최적 시장**: 명확한 가격 범위 내에서 등락하는 횡보장
- **부적합 시장**: 강한 상승/하락 추세 (그리드 범위 이탈 위험)

## 8. 브레이크아웃 모멘텀 (Breakout Momentum)

> 파일: `breakout_momentum.py` | 시장: **전체** | 난이도: 쉬움

### 원리

터틀 트레이딩의 핵심 전략. N일간의 최고가를 돌파하면 강한 모멘텀이 발생했다고 판단하여 매수. 거래량이 동반되면 신뢰도가 높아진다. 승률은 낮지만(30~40%) 한 번의 큰 수익으로 여러 번의 작은 손실을 상쇄하는 구조.

### 매수/매도 조건

```
매수: 현재가 > 최근 20일 최고가 (거래량 1.5배 이상 동반 시 신뢰도 상승)
매도: 현재가 < 최근 10일 최저가
```

### 시각화

```
가격
 │
 │                    ╱ ← 돌파! (매수)
 │ ═══════════════════    ← 20일 최고가
 │    ╱╲   ╱╲    ╱╲ ╱
 │   ╱  ╲ ╱  ╲  ╱  ╳
 │  ╱    ╳    ╳ ╱
 │ ╱           ╳
 │
 │ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  ← 10일 최저가 (매도선)
 └──────────────────────── 시간
```

### 파라미터

| 파라미터 | 기본값 | 범위 | 설명 |
|---------|-------|------|------|
| `entry_period` | 20 | 10~30 | 매수 돌파 판단 기간 |
| `exit_period` | 10 | 5~15 | 매도 이탈 판단 기간 |
| `volume_filter` | true | - | 거래량 필터 사용 여부 |

### 시장 평가

- **강점**: 큰 추세의 시작을 포착, 규칙이 명확, 역사적으로 검증된 전략
- **약점**: 승률 자체는 낮음 (30~40%), 가짜 돌파(fake-out)에 취약
- **최적 시장**: 추세 시작 구간, 코인의 폭발적 상승 직전
- **부적합 시장**: 좁은 범위의 횡보장 (가짜 돌파 빈번)

## 9. 볼린저 스퀴즈 (Bollinger Squeeze)

> 파일: `bollinger_squeeze.py` | 시장: **횡보→추세 전환** | 난이도: 중간

### 원리

볼린저 밴드의 폭이 극도로 좁아지면(스퀴즈) 곧 큰 변동성 폭발이 올 가능성이 높다는 원리. 코인 시장은 변동성 축소 후 폭발적 움직임이 자주 나타나므로 유용하다.

### 매수/매도 조건

```
매수: 밴드 폭이 최근 120일 중 최소 수준
      + 가격이 상단 밴드를 돌파
      + 밴드가 확장되기 시작

매도: 모멘텀 약화 (밴드가 다시 수축 시작) + 수익 중일 때
```

### 시각화

```
밴드폭
 │
 │  넓음           넓음
 │  ╲             ╱
 │   ╲           ╱   ← 밴드 확장 = 폭발!
 │    ╲    ╱╲   ╱
 │     ╲  ╱  ╲ ╱
 │      ╲╱    ╳ ← 스퀴즈 (최소)
 │             ↑
 │        여기서 방향 확인 후 진입
 └──────────────────────── 시간
```

### 파라미터

| 파라미터 | 기본값 | 범위 | 설명 |
|---------|-------|------|------|
| `bb_period` | 20 | 15~25 | 볼린저 밴드 기간 |
| `bb_std` | 2.0 | 1.5~2.5 | 표준편차 배수 |
| `squeeze_lookback` | 120 | 60~200 | 스퀴즈 판단 기간 (길수록 보수적) |

### 시장 평가

- **강점**: 큰 움직임의 시작을 포착, 리스크 대비 수익률이 높음
- **약점**: 스퀴즈 발생 빈도가 낮음 (대기 시간 김), 방향 판단이 어려움
- **최적 시장**: 횡보 후 추세 전환이 예상되는 구간
- **부적합 시장**: 이미 강한 추세가 진행 중인 시장

## 전략 선택 가이드

### 시장 상태별 권장 전략

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  상승장 (bullish)                                    │
│  ├── 1순위: 변동성 돌파 (단기, 일중 매매)              │
│  ├── 2순위: 슈퍼트렌드 (중기, 추세 추종)              │
│  └── 3순위: MACD / MA 교차 (중장기, 큰 추세)          │
│                                                     │
│  횡보장 (sideways)                                   │
│  ├── 1순위: 그리드 트레이딩 (자동 분할 매매)            │
│  ├── 2순위: 볼린저 밴드 (밴드 반전)                    │
│  ├── 3순위: RSI 평균 회귀 (과매수/과매도)              │
│  └── 주목: 볼린저 스퀴즈 (추세 전환 대비)              │
│                                                     │
│  하락장 (bearish)                                    │
│  └── 매매 중단 (현금 대기)                            │
│                                                     │
│  추세 시작 구간                                       │
│  └── 브레이크아웃 모멘텀 (N일 최고가 돌파)              │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 전략 조합 원칙

1. **같은 방향의 전략을 조합**하라. 추세 추종 + 평균 회귀를 동시에 돌리면 서로 상충한다.
2. **시장 상태 감지가 핵심**이다. 전략 자체보다 "지금이 어떤 장인가"를 정확히 판단하는 것이 더 중요하다.
3. **손절은 모든 전략의 전제 조건**이다. 어떤 전략이든 손절 없이 운용하면 한 번의 큰 손실로 전체 수익을 날릴 수 있다.
4. **백테스트 결과를 맹신하지 말라**. 과최적화(overfitting) 위험. 충분한 Out-of-Sample 테스트 필수.

## 코드에서 전략 사용하기

```python
from cryptobot.strategies import StrategyRegistry
from cryptobot.strategies.base import StrategyParams
from cryptobot.strategies.volatility_breakout import VolatilityBreakout
from cryptobot.strategies.rsi_mean_reversion import RSIMeanReversion
from cryptobot.strategies.bollinger_bands import BollingerBands

# 1. 전략 등록
registry = StrategyRegistry()
registry.register(VolatilityBreakout(StrategyParams(extra={"k_value": 0.5})))
registry.register(RSIMeanReversion())
registry.register(BollingerBands())

# 2. 시장 상태로 자동 선택
strategy = registry.select_by_market("bullish")  # → 변동성 돌파
strategy = registry.select_by_market("sideways")  # → RSI 평균 회귀

# 3. 이름으로 직접 선택
strategy = registry.get("bollinger_bands")

# 4. 매수/매도 신호 확인
signal = strategy.check_buy(df, current_price)
if signal.signal_type == "buy":
    print(f"매수! 신뢰도: {signal.confidence}, 사유: {signal.reason}")
```
